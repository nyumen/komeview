# komeview（NCOverlay デスクトップ版） — 要件定義 (SPEC)

アプリ名: **komeview**。
ブラウザ拡張「NCOverlay」を土台に、**任意の動画（ネイティブVODアプリ / ローカル動画 / フルスクリーン動画など）の上にニコニコ風コメントを重ねて表示する**デスクトップアプリ。
コメ専アプリ「commenomi」のUI/操作感を踏襲しつつ、独自機能を追加する。

- 技術スタック: Electron + React + Vite + `@xpadev-net/niconicomments`
- 配置: 本リポジトリの `desktop/` 配下（ブラウザ拡張は `src/` 配下のまま）

---

## 1. コンセプトと前提

- ブラウザ拡張版は動画プレイヤーのDOMから再生位置を読めるため**自動同期**できるが、デスクトップ版は外部アプリの再生位置を読めない。
- そのため**仮想クロック（手動同期）**でコメントを再生する。ユーザーが頭出しを合わせ、シーク/速度で調整する前提。
- 透明な常時最前面ウィンドウを動画の上に重ねて使う。

---

## 2. ウィンドウ構成

**1枚のウィンドウ**で構成する。

- 透明（`transparent: true`）・フレームレス（`frame: false`）・**常時最前面**（always on top）
- 自前タイトルバー
  - ファイル名を表示
  - ドラッグでウィンドウ移動（`-webkit-app-region: drag`）
  - 最小化 / 閉じる
- 中央: コメント描画 canvas（透明 = 背後の動画が見える / niconicomments で描画）

### 2.1 ウィンドウの枠とレイアウト
- **ウィンドウモードでは枠（ウィンドウの境界線）を表示する。** ウィンドウがどこにあるか分かるようにするため。
  - 透明・フレームレスなので、枠はCSSのボーダー等で表現する（ネイティブのウィンドウ枠は透過と両立しないため使わない）。
- **コメント描画 canvas はタイトルバーの高さ分だけ下げる**。ウィンドウモードでタイトルバーと流れるコメントが重ならないようにする。
- 擬似全画面モードでは枠・タイトルバーともに**非表示**にし、canvas を全面に描画する（§2.3）。

### 2.2 フルスクリーン最前面のOS依存（要検証）
- フルスクリーン排他モードの動画の上に乗らないことがある（特に macOS のフルスクリーン Space、Windows のゲーム系全画面）。window level の調整が必要。**早期に検証する。**

### 2.3 擬似全画面モード（commenomi 再現）
画面いっぱいにコメントを重ねるモード。**OS の本物の全画面（maximize / `setFullScreen`）は使わない。**

- **問題**: 本物の全画面・常時最前面にすると Netflix 等の動画再生が真っ黒になる。
- **解決（擬似全画面）**: ウィンドウを「乗っているディスプレイ全体より上下左右1pxずつ小さいサイズ」にリサイズし、位置を固定する。
  - 対象ディスプレイ = `screen.getDisplayMatching(現在のbounds)`
  - bounds = `{ x: display.bounds.x + 1, y: display.bounds.y + 1, width: display.bounds.width - 2, height: display.bounds.height - 2 }`
  - （workArea ではなくディスプレイ全体 `bounds` を基準にする）
- **トリガー**:
  - コメント領域の**ダブルクリック**で擬似全画面に入る。
  - 右クリックメニューの「全画面」でも入る。
  - もう一度ダブルクリック（または同メニュー）で**元のウィンドウサイズ/位置に戻す**（直前の通常bounds を保存しておき復元）。
  - **`Esc` キーでも解除**できる。右クリックメニューの「全画面」トグルからも解除できる。
- **擬似全画面中の見た目/挙動**:
  - 枠・自前タイトルバーを非表示にし、ウィンドウの境界が一切見えないようにする。
  - 位置を固定（ドラッグ移動不可）。
  - 下部バーの自動表示/非表示は通常どおり維持。

### 2.4 ウィンドウ背景
コメントの視認性を上げるため、ウィンドウ背景（コメント領域）の色を選べる。右クリックメニューから変更（→ §9）・永続化（→ §10）。**デフォルトは透明。**

- `透明`（デフォルト）/ `グレー 25%` / `グレー 50%` / `グレー 75%` / `黒` の5択。
- グレー・黒は**不透明の単色**。%は黒に近づく濃さの段階を表す（25%→`#bfbfbf` / 50%→`#808080` / 75%→`#404040`）。
- 実装はコメント描画領域（`.app`）の CSS `background` に反映。

## 3. 下部バー（操作パネル）

通常は**マウスが下端付近に近づいた時だけ表示**し、離れると一定時間後にフェードアウトする。
ただし右クリックメニューの「操作パネルを常時表示」がONのときは**常に表示**する（§9, §10）。

- 再生 / 一時停止
- 停止
- 前マーカー / 次マーカー
- 速度表示（例 `2x`）→ **クリックで速度メニュー**（→ §6）
- シークバー（→ §5）＋ コメント密度ヒストグラム（→ §8）
- 時間表示（現在 / 総時間）
- **次マーカーへの「ジャンプ」ボタン＋残り時間表示**（→ §3.1）
- コメント統計 `total / max / avg`（→ §8）

### 3.1 次のジャンプ先の表示（独自機能）
時間表示の隣に「ジャンプ」ボタンと、その右に次のジャンプ先までの残り時間を表示する。

- **次のジャンプ先** = 現在の再生位置より後にある最初のマーカー（時系列で次のもの）。
  - 例: OP 再生中（OP と A の間）なら次は `A`。
- 表示形式: `"[ラベル]"まで残り mm:ss`（ラベルは `ｷﾀ / OP / A / B / C / ED`）。
  - 残り時間 = 次マーカー時刻 − 現在時刻。**リアルタイムで更新**する（再生中はカウントダウン）。
- 「ジャンプ」ボタンを押すと、その次マーカーへジャンプする。
- 次のマーカーが無い場合（最後のマーカーより後 / マーカー未検出）は、ボタンを無効化し残り時間表示は出さない。

## 4. キーボードショートカット

**ローカルのみ**（このアプリがアクティブな時だけ有効）。**グローバルショートカットは使わない。**

| キー | 動作 |
|---|---|
| `Space` | 再生 / 一時停止 |
| `k` | マーカー「ｷﾀ」へジャンプ |
| `o` | マーカー「OP」へジャンプ |
| `a` | マーカー「A」へジャンプ |
| `b` | マーカー「B」へジャンプ |
| `c` | マーカー「C」へジャンプ |
| `e` | マーカー「ED」へジャンプ |
| `j` | 次のマーカーへジャンプ（操作パネルの「ジャンプ」ボタンと同じ / §3.1） |
| `→ / ←` | +1秒 / −1秒（**固定・変更不可**） |
| `↑` | 先送り（デフォルト 15秒・設定で変更可） |
| `↓` | 巻き戻し（デフォルト 15秒・設定で変更可） |
| `Esc` | 擬似全画面を解除（§2.3） |

マーカーのジャンプ先算出は既存の `src/shared/markers.ts`（`findMarkers`）をそのまま使用する。

### 4.1 スクラブシーク（調整中の自動一時停止）
`←→` / `↑↓` / 操作パネルの `≪ ≫` / マウスホイールによるシーク調整では、
**調整中は再生を一時停止**し、**操作が止んで 500ms 後に（元々再生中だった場合のみ）自動で再生を再開**する。
`Space` や停止ボタンなど明示的な再生/停止操作は、この自動再開の予約より優先される。

### 4.2 マウスホイールシーク
コメント領域上のマウスホイールでシークできる（1ノッチ ≒ 1秒、トラックパッドは delta に比例してスムーズ）。
スクロール中は §4.1 のスクラブ動作（一時停止→自動再開）に従う。メニュー表示中は無効。

### 4.3 自動再生
XML を読み込んだら（ドラッグ&ドロップ / ファイル選択 / アイコンドロップ起動のいずれでも）、**先頭から自動で再生を開始**する。

## 5. シークバー

- ドラッグ / クリックでシーク。
- **ホバー時刻表示**: シークバー上にカーソルを置くと、その位置の時刻をツールチップ表示（commenomi 再現）。
  - 時刻 = `(カーソルX / バー幅) × duration`
  - ※ ホバー位置の「勢い」数値は表示しない（時刻のみ）。
- **マーカー点表示（独自機能）**: 検出できた各マーカーの中心時刻を点として表示。
  - 位置 = `(markerMs/1000 / duration) × バー幅`
  - 該当コメントなしのマーカーは表示しない。
  - 点は**シークバーのトラックに重ねて**表示する。
  - `ｷﾀ / OP / A / B / C / ED` のラベルを点の上に**常時表示**（半透明の控えめな背景）。
    右クリックメニューで「マウスオーバー時のみ表示」へ切替可・永続化（デフォルトは常時表示）。
  - 点をクリックでそのマーカーへジャンプ（キーボードと同動作）。
  - マーカー種別ごとに点を色分け（種類をホバーなしでも判別可能に）。
  - 下部バーが表示されている間のみ見える（自動表示/非表示に追従）。

## 6. 速度メニュー

下部バーの速度表示をクリックで開く。**自前のHTMLポップアップ**で実装（透明ウィンドウ上での統一感のためOSネイティブメニューは使わない）。添付参考画像の質感を再現。

- `現在 2.00倍`（見出し / 現在値を表示）
- `少し速く (+0.1倍)`
- `少し遅く (-0.1倍)`
- プリセット（高い順・現在値にチェック表示）:
  - `2.0倍 / 1.75倍 / 1.5倍 / 1.25倍 / 1.0倍 (標準速度) / 0.75倍`

## 7. コメント表示（フォントサイズ・不透明度）

ブラウザ拡張版にある表示調整を移植する。いずれも右クリックメニューから変更（→ §9）・永続化（→ §10）。

### フォントサイズ
- **デフォルト 100%**（=最大）。**最小 50%** まで、**10%刻み**（`100% / 90% / 80% / 70% / 60% / 50%`）。
- 実装は `@xpadev-net/niconicomments` の `scale`（globalScale）オプションで倍率を反映。

### 不透明度
- 流れるコメントの不透明度。**デフォルト 100%**、**最小 50%** まで、**10%刻み**（`100% / 90% / 80% / 70% / 60% / 50%`）。
- 実装はコメント描画 canvas の CSS `opacity` で反映（コメント層全体を半透明にする）。

### 大量コメント対策（20MB級XMLでの固まり・クラッシュ対応）
- **lazy レンダリング（条件付き）**: 描画コメント数が **30,000件（`LAZY_THRESHOLD`）を超える場合のみ** niconicomments の `lazy: true` を有効化。Y座標（衝突）計算を描画時に分割払いし、読み込み時のフリーズを防ぐ。
  - lazy の副作用としてシーク直後の位置解決が一瞬重くなるため、通常サイズのファイルでは eager（事前計算）のままにする。
- **再描画スキップ**: 整数 vpos が前回描画時と同じ場合は `drawCanvas` を呼ばない（一時停止中のCPU削減）。
- **コメント最大表示数（描画のみの間引き）**: 1秒バケットごとに最大 N 件を等間隔サンプリング。
  - 右クリック「コメント最大表示数」: `制限なし（デフォルト） / 秒間 100件 / 50件 / 25件`。永続化。
  - **統計（勢いグラフ・total/max/avg）とマーカー検出は間引き前の全コメント**から計算し、間引くのは描画へ渡す分だけ。
  - niconicomments はコメント×表示時間ぶんのタイムライン索引を（lazyでも）読み込み時に構築するため、数十万コメント級ではこの索引がメモリを圧迫しクラッシュの原因になり得る（メモリの少ないマシンで顕在化）。その場合の根本対策がこの間引き。

## 8. コメントの勢い（density / コメ/分）

### アルゴリズム: スライディングウィンドウ（コメ/分換算）
固定バケットではなく、各時刻 `t` を中心に幅 `W` の窓でコメント数を数え、分換算する。

```
勢い(t) = (区間 [t - W/2, t + W/2] のコメント数) × (60 / W)
```

- 固定バケットを採用しない理由: ビン境界のエイリアシング / アラインメント依存 / 階段状のカクつき。
- ソート済みコメントに対し two-pointer で O(N) 算出。

### 用途ごとの窓幅
| 用途 | 窓 | 備考 |
|---|---|---|
| 勢い波形グラフ | 中心化・W=10秒 | 応答よく描画 |
| `max` | スライディング・60秒 | 「コメ/分」の指標として安定 |

※ シークバーのホバー位置に勢い数値を出す機能は**設けない**（§5）。

### 角の統計
- `total` = 総コメント数
- `avg` = 総コメント数 ÷ 総時間（分）
- `max` = 60秒スライディング窓の勢いの最大値

### 波形グラフの描画
描画幅のピクセル数ぶんビンを用意し、各ビンを上記スライディング窓のコメ/分で高さにする（または1秒ビンのカウントを軽く平滑化）。下部バーのシークバー上に重ねて表示する。

## 9. 右クリック コンテキストメニュー

**自前のHTMLポップアップ**で実装。

- XMLファイルを開く
- 全画面（擬似全画面）ON / OFF（→ §2.3）
- フォントサイズ（サブメニュー / 100%〜50% を10%刻み / §7）
- 不透明度（サブメニュー / 100%〜50% を10%刻み / §7）
- 大きいシーク量の変更（↑↓、デフォルト 15秒 / `3 / 5 / 10 / 15 / 30`）
- 背景（サブメニュー / 透明・グレー25/50/75%・黒 / §2.4）
- コメント最大表示数（サブメニュー / 制限なし・秒間100・50・25件 / §7）
- 操作パネルを常時表示 ON / OFF（→ §3）
- マーカーラベルを常時表示 ON / OFF（→ §5 / デフォルト ON）

※ 速度変更は下部バーの速度メニュー（§6）にあるため、右クリックメニューには重複して置かない。
- 最前面 ON / OFF
- クリック透過 ON / OFF
- 閉じる

### クリック透過について
透明な1枚ウィンドウのため、コメント領域のクリックは（背後の動画ではなく）このアプリに当たる。動画側を操作したい時のために**クリック透過トグル**を用意する（ON の間は `setIgnoreMouseEvents(true, { forward: true })` でマウスが全部背後へ抜ける）。
グローバルショートカットを使わない制約のため、ON 中もタイトルバー/下部バー上にカーソルが来た時だけ操作可能になるよう動的に切り替える（メニューから解除できるようにするため）。

## 10. 設定の永続化

- 保存先: `app.getPath('userData')/settings.json`（軽量なので `electron-store` 等は使わず自前で読み書き）。
- 保存項目:
  - 速度
  - フォントサイズ倍率（§7）
  - コメント不透明度（§7）
  - ウィンドウ位置 / サイズ（`move` / `resize` / 終了時に保存）
  - 擬似全画面モードの ON / OFF（§2.3。起動時に bounds を再計算して復元する）
  - 大きいシーク量（↑↓、デフォルト 15秒）
  - ウィンドウ背景（§2.4）
  - 操作パネルの常時表示 ON / OFF（§3）
  - マーカーラベルの常時表示 ON / OFF（§5）
  - コメント最大表示数（秒間の最大表示件数 / §7）
  - 最前面 ON / OFF の状態
  - クリック透過 ON / OFF の状態
- 起動時に復元。

### 将来構想：コメントリストパネル（NCOverlay のサイドパネル相当）

- 微調整シーク（←→）は **±1秒固定**とし、量の変更機能は廃止。
- ウィンドウモードで枠を表示し、canvas をタイトルバー分下げる（§2.1）。
- 操作パネルの常時表示オプションを追加（§3, §9, §10）。
- 操作パネルに「次のジャンプ先までの残り時間表示＋ジャンプボタン」を追加（§3.1）。
- コメントの不透明度調整（100%〜50%・10%刻み・永続化）を追加（§7）。
- 右クリックメニューの「再生 / 一時停止」項目を削除（Space と操作パネルのボタンで操作）。
- アプリ名を **komeview** に変更。
- 操作パネルの `≪` `≫` を「マーカージャンプ」から「±1秒シーク（←→と同じ）」に変更。
- 右クリックメニューの「速度」を削除（下部バーと重複のため / §9）。
- 速度プリセットを6個に整理（`2.0 / 1.75 / 1.5 / 1.25 / 1.0 / 0.75`）。内部の倍率丸めを小数2桁に修正（1.25/0.75 等が正しく扱えるように）（§6）。
